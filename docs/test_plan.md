## 5. Bug记录与跟踪流程

### 5.1 Bug记录格式

| 字段 | 说明 | 示例 |
|------|------|------|
| Bug ID | 唯一标识 | BUG-001 |
| 模块 | 所属模块 | game_engine.py |
| 类型 | 错误类型 | 逻辑错误/数据错误/边界错误 |
| 优先级 | P0-P3 | P0(致命)/P1(严重)/P2(一般)/P3(轻微) |
| 状态 | 状态 | 待修复/修复中/已修复/已验证/关闭 |
| 描述 | 问题描述 | 模式3平局时积分计算错误 |
| 重现步骤 | 复现方法 | 1. 进入模式3 2. 连续平局5次 |
| 预期结果 | 正确行为 | 平局5次后+1积分 |
| 实际结果 | 当前行为 | 未增加积分 |
| 发现人 | 测试人员 | Tester |
| 发现日期 | 日期 | 2026-06-01 |
| 修复人 | 开发人员 | Coder |
| 修复日期 | 日期 | - |

### 5.2 Bug处理流程

```
发现Bug
  │
  ▼
┌─────────────────┐
│ 记录Bug信息     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 优先级评估      │
│ P0/P1/P2/P3    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 分配给开发人员  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 开发修复        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 测试验证        │
└────────┬────────┘
         │
    通过 │ 失败
    │    │
    ▼    ▼
  关闭  重新修复
```

### 5.3 Bug优先级定义

| 优先级 | 定义 | 处理时限 |
|--------|------|----------|
| P0 | 致命错误，导致系统崩溃或核心功能不可用 | 立即修复 |
| P1 | 严重错误，影响主要功能正常使用 | 24小时内 |
| P2 | 一般错误，影响部分功能或用户体验 | 72小时内 |
| P3 | 轻微错误，不影响功能使用 | 下一版本修复 |
