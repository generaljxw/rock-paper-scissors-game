## 🎯 项目目标

为10岁以下儿童开发一款有趣的猜拳游戏，具有卡通元素和社交属性，支持人机对战和玩家间对战。

## 🌟 特性需求列表

### 🎮 核心玩法特性 (MVP)

| ID | 特性名称 | 描述 | 优先级 | 来源 |
|----|----------|------|--------|------|
| SF-001 | 用户登录 | 用户登录后才能进行对战，保存用户对战记录，支持查看历史对战记录 | **P0** | 需求文档 |
| SF-002 | 人机对战 | 玩家与AI进行猜拳对战 | **P0** | 需求文档 |
| SF-003 | 对战模式选择 | 三种对战模式：<br>**模式1（一局定胜负）**：打平继续出拳，分胜负后结算<br>**模式2（三局两胜）**：每局必须分出胜负（平局则继续出拳），先获2胜者获胜；当前两局同一方获胜时，无需第三局<br>**模式3（连战模式）**：胜利/平局可继续出拳，失败则结算 | **P0** | 需求文档 |
| SF-004 | 胜负判定 | 根据标准猜拳规则判定胜负（石头>剪刀>布>石头） | **P0** | 需求文档 |
| SF-005 | 积分系统 | 记录玩家总积分，不同模式规则：<br>**模式1**：不耗积分，胜利+1分，失败-1分<br>**模式2**：不耗积分，胜利+2分，失败-2分<br>**模式3（连战模式）**：需消耗3积分进入；连续胜利超过3次奖励+5分（出拳相同不计入）；每累计5次出拳额外+1分<br>**积分检查**：连战模式"再玩一局"时需再次检查积分是否≥3，不足则提示"积分不足，无法继续对战" | **P0** | 需求文档 |
| SF-006 | 胜率统计 | 统计玩家平均胜率，以每次选择对战模式后的对战结果为统计单位 | **P0** | 需求文档 |
| SF-006a | 战绩记录 | 战绩以每轮对战结算后的汇总信息为单位记录，仅包含4个字段：对战时间、游戏模式、对战结果、积分变化。移除玩家和AI的具体出拳记录。仅在对战正常结算后生成记录，中途退出或积分不足时不产生战绩记录 | **P0** | 需求文档 |
| SF-007 | 结算画面 | 显示对战结果、动画效果 | **P0** | 需求文档 |
| SF-008 | 卡通界面 | 儿童友好的卡通风格UI设计 | **P0** | 需求文档 |
