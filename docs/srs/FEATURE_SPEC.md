## 🎯 项目目标

为10岁以下儿童开发一款有趣的猜拳游戏，具有卡通元素和社交属性，支持人机对战和玩家间对战。

## 🎵 SF-022 需求拆解 (SR)

### SR1：游戏登录界面播放背景音乐

**功能（主流程）**

1. 用户打开游戏应用程序，系统成功加载并展示登录界面后，立即自动播放预设的登录界面背景音乐；
2. 用户正确输入账号密码并验证通过后，登录界面背景音乐应立即停止播放，并自动无缝切换至模式选择界面背景音乐；
3. 用户从游戏内其他界面执行退出登录操作并成功返回至登录界面时，系统应自动播放登录界面背景音乐。

**功能（分支流）**

1a. 用户在登录界面保持静止状态且无任何操作达到预设时长阈值时，主界面背景音乐应持续循环播放，无中断或变化；

2a. 用户登录验证失败（包括但不限于账号密码错误、网络连接异常等情况）时，登录界面背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

3a. 用户主动关闭游戏应用程序（包括正常退出和异常关闭）时，所有音乐播放应立即停止。

**可服务性**

音乐风格悠扬、舒缓；所有音乐素材必须具备较高的艺术美感和品质感，不得使用音质低劣、廉价感明显的音频素材

**规格**

音乐时长不低于30秒，单个音乐文件应控制在3MB以下，避免程序占用空间膨胀

---

### SR2：模式选择界面播放背景音乐

**功能（主流程）**

1. 用户从登录界面成功登录后进入模式选择界面时，系统应立即自动播放模式选择界面背景音乐，与登录界面背景音乐实现无缝切换；
2. 用户在模式选择界面点击任一对战模式的"开始对战"按钮并成功进入对战界面时，模式选择界面背景音乐应立即停止播放，并自动无缝切换至对战界面背景音乐；
3. 用户在模式选择界面点击"查看战绩"按钮进入战绩查询界面时，模式选择界面背景音乐应立即停止播放；
4. 用户从对战界面或战绩查询界面返回模式选择界面时，系统应自动恢复播放模式选择界面背景音乐；
5. 用户在模式选择界面点击"退出登录"按钮并确认退出后，模式选择界面背景音乐应立即停止播放，并自动无缝切换至登录界面背景音乐。

**功能（分支流）**

1a. 用户在模式选择界面保持静止状态且无任何操作达到预设时长阈值时，背景音乐应持续循环播放，无中断或变化；

2a. 用户选择连战模式但积分不足（弹出"积分不足"提示框）时，模式选择界面背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

2b. 用户选择连战模式弹出"确认入场"对话框后点击"否"取消时，模式选择界面背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

3a. 用户从战绩查询界面返回模式选择界面时，系统应自动恢复播放模式选择界面背景音乐，从曲目起始位置重新开始播放；

4a. 用户退出登录弹出"确认退出"对话框后点击"否"取消时，模式选择界面背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

5a. 用户主动关闭游戏应用程序（包括正常退出和异常关闭）时，所有音乐播放应立即停止。

**可服务性**

音乐风格轻松、明快，体现游戏选择与期待感；三种对战模式的卡片区域视觉风格分别为热情红（#FF6B6B）、清新绿（#4ECDC4）、活力黄（#FFE66D），背景音乐需与整体界面的卡通友好风格高度贴合；所有音乐素材必须具备较高的艺术美感和品质感，不得使用音质低劣、廉价感明显的音频素材

**规格**

音乐时长不低于30秒，单个音乐文件应控制在3MB以下，避免程序占用空间膨胀

---

### SR3：对战界面播放背景音乐和音效

**功能（主流程 - 背景音乐）**

1. 用户从模式选择界面成功进入对战界面时，系统应立即自动播放对战界面背景音乐，与模式选择界面背景音乐实现无缝切换；
2. 用户在对战界面每回合选择出拳（石头/剪刀/布）时，系统应自动将背景音乐音量适当调低，确保音量不会过低，以能够清晰突出出拳及相关操作音效为标准，同时维持背景音乐的存在感；
3. 出拳回合结果判定完毕并进入下一回合出拳准备阶段时，系统应自动将背景音乐音量恢复至正常水平；
4. 对战结束进入结算阶段时，对战界面背景音乐应立即停止播放，并自动无缝切换至结算背景音乐；
5. 用户点击"返回主界面"按钮离开对战界面时，对战界面背景音乐应立即停止播放，并自动无缝切换至模式选择界面背景音乐。

**功能（主流程 - 音效）**

6. 用户在对战界面点击出拳按钮（石头/剪刀/布）时，系统应立即播放对应的出拳音效，出拳音效应在用户点击后100毫秒内触发播放；
7. 每回合出拳结果判定后，系统应根据胜负结果播放对应的判定音效：胜利音效或失败音效；
8. 判定音效播放完毕后，若对战未结束（进入下一回合），系统应在下一回合出拳提示出现时恢复出拳按钮的可交互状态，此时音效已播放完毕。

**功能（分支流 - 背景音乐）**

1a. 用户在对战过程中，背景音乐应持续循环播放，回合切换期间不中断、不重置；

2a. 三种对战模式（一局定胜负、三局两胜、连战模式）的背景音乐应保持整体游戏音乐风格的统一性，同时在音乐风格、节奏、氛围上保持适当的差异化，体现各自特色：一局定胜负节奏明快紧凑，三局两胜节奏稳健有层次感，连战模式节奏激烈且富有推进感；

3a. 用户点击"再来一局"按钮重新开始对战时，对战界面背景音乐应从曲目起始位置重新开始播放；

4a. 用户选择连战模式但积分不足（弹出"积分不足"提示框）时，对战界面背景音乐保持当前播放状态，继续循环播放，不发生任何变化。

**功能（分支流 - 音效）**

5a. 用户点击出拳按钮后，出拳按钮应立即变为不可用状态（禁用），出拳音效在禁用状态期间播放，音效播放期间用户不可重复触发出拳；

6a. 连战模式中玩家战败时的失败音效与普通失败音效保持统一标准，无需额外区分；

6b. 连战模式中玩家连续胜利达到3次及以上时的胜利音效与普通胜利音效保持统一标准，无需额外区分；

7a. 平局时系统自动进入下一回合，平局音效播放完毕后自动恢复出拳按钮的可交互状态，无需用户额外操作；

8a. 用户主动关闭游戏应用程序（包括正常退出和异常关闭）时，所有音乐和音效播放应立即停止。

**可服务性**

背景音乐风格紧张、竞技，体现对决氛围；出拳音效需短促有力，判定音效需明确区分胜负两种状态；三种模式的背景音乐需在保持整体风格统一性的基础上，通过风格、节奏、氛围的适当差异化体现模式特色；所有音乐和音效素材必须具备较高的艺术美感和品质感，不得使用音质低劣、廉价感明显的音频素材

**规格**

背景音乐时长不低于30秒，单个背景音乐文件应控制在3MB以下；单个音效文件时长应控制在1-3秒，音效文件大小应控制在500KB以下；所有音乐和音效文件总大小应控制在15MB以下，避免程序占用空间膨胀

### SR4：结算界面播放背景音乐

**功能（主流程）**

1. 每轮游戏结束并进入结算阶段时，系统应自动播放结算背景音乐，与对战界面背景音乐实现无缝切换；
2. 结算背景音乐应根据最终胜负结果播放不同风格的音乐：胜利时播放欢快、庆祝风格的音乐，失败时播放低沉、鼓励风格的音乐；
3. 连战模式的胜利结算音乐和失败结算音乐需与其他两种模式（一局定胜负、三局两胜）的结算音乐有明显区分度；
4. 用户点击"再来一局"按钮重新开始对战时，结算背景音乐应立即停止播放，并自动无缝切换至对战界面背景音乐；
5. 用户点击"返回主界面"按钮时，结算背景音乐应立即停止播放，并自动无缝切换至模式选择界面背景音乐。

**功能（分支流）**

1a. 结算背景音乐在播放期间应持续循环，直到用户执行离开结算阶段的操作（点击"再来一局"或"返回主界面"）为止；

2a. 连战模式中玩家连续胜利达到3次及以上并获得额外奖励积分时，胜利结算音乐与连战模式普通胜利结算音乐保持一致，无需额外区分；

3a. 用户点击"再来一局"按钮后，若为连战模式且积分不足（弹出"积分不足"提示框）时，结算背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

3b. 用户点击"再来一局"按钮后，若为连战模式弹出"确认入场"对话框后点击"否"取消时，结算背景音乐保持当前播放状态，继续循环播放，不发生任何变化；

4a. 用户主动关闭游戏应用程序（包括正常退出和异常关闭）时，所有音乐播放应立即停止。

**可服务性**

结算音乐需与对战结果高度契合：胜利音乐传递成就感与喜悦，失败音乐传递鼓励而非挫败感；音乐风格需与游戏整体卡通友好风格一致；连战模式的结算音乐需在保持整体品质感的基础上，与其他模式形成明显区分度；所有音乐素材必须具备较高的艺术美感和品质感，不得使用音质低劣、廉价感明显的音频素材

**规格**

普通模式（一局定胜负、三局两胜）需胜利和失败各1首结算音乐，共2首；连战模式需胜利和失败各1首结算音乐，共2首，且需与普通模式结算音乐有明显区分度；合计共4首结算音乐；每首音乐时长不低于15秒，单个音乐文件应控制在3MB以下
